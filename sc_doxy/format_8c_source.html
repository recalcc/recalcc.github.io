00014  *
00015  * The format function will produce a string representation of a number
00016  * given a _format_ (described below) and a double value.  The result is
00017  * written into the passed buffer -- if the resulting string is too
00018  * long to fit into the passed buffer, the function returns false.
00019  * Otherwise the function returns true.
00020  *
00021  * The fmt parameter contains the format to use to convert the number.
00022  *
00023  *  #   Digit placeholder.  If the number has fewer digits on either
00024  *      side of the decimal point than  there are '#' characters in
00025  *      the format, the extra '#' characters are ignored.  The number
00026  *      is rounded to the number of digit placeholders as there are
00027  *      to the right of the decimal point.  If there are more digits
00028  *      in the number than there are digit placeholders on the left
00029  *      side of the decimal point, then those digits are displayed.
00030  *
00031  *  0   Digit placeholder.  Same as for '#' except that the number
00032  *      is padded with zeroes on either side of the decimal point.
00033  *      The number of zeroes used in padding is determined by the
00034  *      number of digit placeholders after the '0' for digits on
00035  *      the left side of the decimal point and by the number of
00036  *      digit placeholders before the '0' for digits on the right
00037  *      side of the decimal point.
00038  *
00039  *  .   Decimal point.  Determines how many digits are placed on
00040  *      the right and left sides of the decimal point in the number.
00041  *      Note that numbers smaller than 1 will begin with a decimal
00042  *      point if the left side of the decimal point contains only
00043  *      a '#' digit placeholder.  Use a '0' placeholder to get a
00044  *      leading zero in decimal formats.
00045  *
00046  *  %   Percentage.  For each '%' character in the format, the actual
00047  *      number gets multiplied by 100 (only for purposes of formatting
00048  *      -- the original number is left unmodified) and the '%' character
00049  *      is placed in the same position as it is in the format.
00050  *
00051  *  ,   Thousands separator.  The presence of a ',' in the format
00052  *      (multiple commas are treated as one) will cause the number
00053  *      to be formatted with a ',' separating each set of three digits
00054  *      in the integer part of the number with numbering beginning
00055  *      from the right end of the integer.
00056  *
00057  *  &   Precision.  When this character is present in the fractional
00058  *      part of the number, it is equavalent to a number of 0's equal
00059  *      to the precision specified in the column format command.  For
00060  *      example, if the precision is 3, "&" is equivalent to "000".
00061  *
00062  *  \   Quote.  This character causes the next character to be
00063  *      inserted into the formatted string directly with no
00064  *      special interpretation.
00065  *
00066  *  E- E+ e- e+
00067  *      Scientific format.  Causes the number to formatted in scientific
00068  *      notation.  The case of the 'E' or 'e' given is preserved.  If
00069  *      the format uses a '+', then the sign is always given for the
00070  *      exponent value.  If the format uses a '-', then the sign is
00071  *      only given when the exponent value is negative.  Note that if
00072  *      there is no digit placeholder following the '+' or '-', then
00073  *      that part of the formatted number is left out.  In general,
00074  *      there should be one or more digit placeholders after the '+'
00075  *      or '-'.
00076  *
00077  *  ;   Format selector.  Use this character to separate the format
00078  *      into two distinct formats.  The format to the left of the
00079  *      ';' character will be used if the number given is zero or
00080  *      positive.  The format to the right of the ';' character is
00081  *      used if the number given is negative.
00082  *    
00083  *  Any
00084  *      Self insert.  Any other character will be inserted directly
00085  *      into the formatted number with no change made to the actual
00086  *      number.
